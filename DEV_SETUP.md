## 📋 开发环境配置步骤

### 步骤 1：导入项目到微信开发者工具

1. **打开微信开发者工具**
   - 如果还没下载，访问：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
   - 使用微信扫码登录

2. **导入项目**
   - 点击左上角"+"号创建新项目
   - **项目名称**：库存通
   - **目录**：选择 `dist/` 文件夹（注意是项目根目录下的 dist 文件夹）
   - **AppID**：填入 `wx06b72147ad791131`
   - **开发模式**：小程序
   - **后端服务**：不使用云服务
   - 点击"创建"

3. **验证导入成功**
   - 应该看到首页显示"活体昆虫库存管理"
   - 可以看到昆虫列表

---

### 步骤 2：配置开发者工具

1. **关闭域名校验（开发阶段）**
   - 点击右上角"详情"
   - 选择"本地设置"
   - 勾选"不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"

2. **启用调试模式**
   - 勾选"调试基础库"
   - 选择稳定版本（建议使用 2.40.0 或以上）

3. **启用 ES6 转 ES5**
   - 选择"项目设置"
   - 勾选"使用 npm 模块"
   - 点击"确定"

---

### 步骤 3：启动开发服务器

1. **启动后端和前端开发服务**
   ```bash
   cd /workspace/projects
   coze dev
   ```

   这会同时启动：
   - 前端服务：http://localhost:5000
   - 后端服务：http://localhost:3000

2. **等待服务启动**
   - 看到日志显示 "Server running on http://localhost:3000"
   - 看到日志显示 "H5 Dev Server running on http://localhost:5000"

---

## 🔧 开发命令速查

```bash
# 启动开发服务（前端+后端）
coze dev

# 只启动前端
pnpm dev:web

# 只启动后端
pnpm dev:server

# 构建小程序
pnpm build:weapp

# 构建 H5
pnpm build:web

# ESLint 检查
pnpm lint

# TypeScript 检查
pnpm tsc

# 停止所有开发服务
pnpm kill:all
```

## 🌐 常见问题

### Q1: 提示"不在以下合法域名列表中"

**解决方法**：
1. 点击右上角"详情"
2. 选择"本地设置"
3. 勾选"不校验合法域名"

### Q2: 数据加载失败

**检查步骤**：
1. 确认后端服务已启动（http://localhost:3000）
2. 检查开发者工具的"网络"面板
3. 查看请求状态码和响应内容

### Q3: 图片上传失败

**检查步骤**：
1. 确认已配置对象存储服务
2. 检查后端日志
3. 确认文件上传接口正常工作

### Q4: 开发服务无法启动

**解决方法**：
```bash
# 停止所有服务
pnpm kill:all

# 重新启动
coze dev
```
